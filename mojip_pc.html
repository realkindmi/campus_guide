마크업 작업규칙

표준 문법의 사용

- HTML은 해당 DTD의 명세에 맞게 작성한다.

- DTD를 제외한 모든 엘리먼트와 속성은 소문자로 작성.

- 속성의 값은 큰따움표("")로 묶는다.

- 특수기호는 Entity name을 사용하여 entity 코드로 변환한다.

Entity Code

Character ₩ " & < >   ·
Entity Name &#8361; &quot; &amp; &lt; &gt; &nbsp; &middot;

파일의 인코딩

*.asp, *.css, *.js의 파일의 저장 방식을 euc-kr로 설정한다.

문자셋(charset)별 인코딩

문자셋(charset) 기본 인코딩
charset=euc-kr ANSI

코드 들여쓰기

- 코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 들여쓰기 규칙을 준수한다.

- 마크업의 중첩이 깊어질 때마다 자식 엘리먼트는 1탭 들여 쓰고, 1탭의 크기는 공백 4칸으로 설정한다.

엘리먼트의 기본 속성 지정 순서

· A

<a href="#" title="새창" target="_blank">버튼1</a> <a href="javascript:alert('추후 오픈 예정입니다.')">버튼2</a>

- alert창을 사용할 경우 href 속성에 자바스크립트로 처리한다

- 앵커 기능이 있다면 해당 요소의 id값을 작성한다.

- 새 창으로 페이지를 표시해야 할 경우 target 속성을 선택적으로 사용한다.

- 링크의 목표가 명확하지 않은 경우 title 속성의 값으로 부가적인 설명을 제공한다.

· Img

<img src="img.gif" alt="대체텍스트"/>

- alt 속성은 반드시 제공되어야 의미가 없는 이미지에 한하여 빈 값을 허용한다.

- 유지보수의 편의성을 위하여 width, height 속성은 사용하지 않는다.
